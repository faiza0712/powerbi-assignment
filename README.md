README – Power BI E-Commerce Data Preparation and Modeling Assignment
Project Title
E-Commerce Sales Data Analysis using Power BI

Objective
The objective of this project is to perform data import, transformation, cleansing, aggregation, and relationship modeling using Power BI and Power Query Editor on an e-commerce sales dataset.

Datasets Used
List of Orders.csv
Order Details.csv
Sales Target.csv
Tasks Performed
Imported all datasets into Power BI and Power Query Editor.

Restricted the List of Orders table to the first 500 rows.

Changed data types for Order Date, Amount, and Target columns.

Formatted Customer Name to Proper Case.

Created a new Location column by merging City and State.

Created a Profit Margin custom column and Profit Status conditional column.

Merged List of Orders and Order Details using Order ID to create the Orders Data table.

Checked for missing values and duplicate records. No missing values or duplicates were found in the dataset.

Sorted Orders Data by Order Date and applied filters for state-wise analysis.

Performed grouping and aggregation on Order Details and Sales Target tables.

Established relationships:

List of Orders ↔ Order Details (Order ID)
Order Details ↔ Sales Target (Category)
Verified relationships using Manage Relationships.

This project demonstrates practical skills in data preparation, transformation, aggregation, and relationship modeling using Power BI, ensuring accurate and business-relevant data analysis.
